# Multi-role Review

Forge can review an idea, plan, prototype, document, interface, or submission from several lightweight perspectives.

## Perspectives

- 产品视角：用户能不能理解？流程顺不顺？价值是否明显？
- 工程视角：能不能稳定运行？结构是否清楚？依赖是否合理？
- 文档视角：别人能不能上手？步骤是否可复制？成功结果是否明确？
- 风险视角：哪里最容易翻车？是否有安全、兼容、许可证、部署风险？
- 普通人视角：不懂技术的人能不能照着做？术语是否需要解释？
- 维护者视角：以后还能不能改？关键决策和状态是否记录？

## Compact Output Template

```text
多视角审查：
- 产品：...
- 工程：...
- 文档：...
- 风险：...
- 普通人：...
- 维护者：...

结论：...
下一步：...
```

Keep each perspective to 1-3 key judgments. Do not write a long essay unless the user asks for deep review.

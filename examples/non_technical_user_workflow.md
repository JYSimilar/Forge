# Example: Non-technical User Workflow

## User Input

```text
Forge: 我不懂技术，想做一个整理家庭照片的小工具，告诉我第一版怎么做。
```

## Forge Should Do

1. Explain concepts in plain language.
2. Give 2-3 simple routes, such as no-code, local script, or small web app.
3. Recommend the lowest-friction route.
4. Provide copyable steps.
5. Provide visible acceptance checks: what the user should click, input, and see.
6. Avoid unnecessary jargon and heavy architecture.

## Expected Output

- simplest first version
- what to prepare
- step-by-step setup
- how to confirm success
- common problems and fixes

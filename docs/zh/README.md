# Forge 中文文档

Forge，中文可以叫“造物”。

一句话：**Forge 是一个帮助普通人通过 AI Agent 把模糊想法做成真实可用成果的 skill。**

它像一个项目经理一样帮你澄清想法、收束第一版 / MVP、审查已有项目、规划工程任务、
把有边界的任务交给当前智能代理、验收结果、修正问题、写文档，并准备交接或发布。

安全任务单是 Forge 管理 AI 执行时使用的默认执行单元，不是 Forge 的全部。
每个 AI 任务都应该有清楚的目标、上下文预算、限制、验收标准、验证证据、停手条件和复查关口。

## 它怎么工作

1. 你说一个想法、需求、项目问题或下一步困惑。
2. Forge 帮你收束目标、拆任务、写清楚边界和验收标准。
3. 当前 agent 按任务单执行，或者你手动复制交接稿给其他工具。
4. 做完后 Forge 帮你验收结果、说明风险、给下一步。

## 当前版本

Forge 2.2.1 是 **发布加固** 补丁版本。

核心原则：

- 默认小步执行。
- 先定义上下文预算，避免过度读取资料。
- 每轮停在复查关口，不自动无限推进。
- 跨工具使用只能由用户手动复制交接稿，Forge 不自动调度其他模型或产品。

## 真实反馈

早期版本已经能减少“AI 偷懒停下”，但在小任务里可能消耗过多 token。

Forge 2.2 把这个反馈转成产品改进：Forge 仍然是项目经理型 skill，
但默认用低 token 的安全任务单来管理 AI 执行，明确上下文预算、停手条件、验收标准和复查关口。

## 流程

```text
模糊想法 / 已有项目 / 个人需求
↓
澄清
↓
选择最小可用版本
↓
规划下一步具体任务
↓
安全任务单
↓
有边界执行
↓
复查关口
↓
文档 / 交接 / 发布
```

## 前后对比

Before:

```text
帮我把这个项目优化一下。
```

After:

```text
目标：
在不大范围重写的前提下，先修最影响交付的问题。

必须读取：
- README
- 运行/测试脚本
- 当前失败现象或用户反馈

不要读取：
- 大型构建产物
- 无关生成文件

限制：
- 不做无关重构
- 不新增重依赖，除非用户确认

验收标准：
- 明确风险被修复或被收束
- 现有行为没有回归

验证方式：
- 运行最小相关测试或手动检查

停手条件：
- 完成一个有边界的改进后停下

复查关口：
- 汇总改动、证据、风险和下一步选项
```

Forge 是项目经理，安全任务单是它管理 AI 执行时使用的工作单元。

## 最小演示

用户输入：

```text
我想做一个能自动整理会议纪要的小工具，第一版怎么开始？
```

Forge 典型输出：

```text
功能目标：
把会议文字整理成结构化纪要，第一版只支持用户粘贴文本。

第一版范围：
1. 输入会议文字
2. 输出摘要、待办事项、负责人、截止时间
3. 支持复制 Markdown

暂时不做：
- 账号系统
- 多人协作
- 自动录音转写
- 日历或飞书集成

给当前智能代理的任务单：
- 目标：搭一个本地可运行的单页原型
- 输入：一段会议文本
- 输出：结构化 Markdown 纪要
- 限制：不接外部 API，不引入复杂后端
- 验收：能用示例文本生成摘要和待办清单
- 验证：运行项目并手动检查输出

下一步：
1. 先做可运行原型
2. 再补 README 和示例输入
3. 最后验收能不能给朋友试用
```

## 高级能力

高级能力默认不会打扰小任务。只有项目变复杂、要交付、要多角色协作、要长期维护时才需要它们。

- Existing Project Audit：接手已有工程前先自查。
- 双索引：生成给人看的 `FORGE_INDEX.md` 和给智能代理/脚本看的 `forge_index.json`。
- Forge Doctor：发布或稳定性总检。
- 多角色协作：当前智能代理内的多角色任务分工。
- 人工交接：用户手动复制给其他工具的交接稿。

# Forge 中文文档

Forge，中文可以叫“造物”。

一句话：**Forge = 把模糊任务变成能交给当前智能代理执行的安全任务单，并负责验收、总结和下一步。**

## 它怎么工作

1. 你说一个想法、需求、项目问题或下一步困惑。
2. Forge 帮你收束目标、拆任务、写清楚边界和验收标准。
3. 当前 agent 按任务单执行，或者你手动复制交接稿给其他工具。
4. 做完后 Forge 帮你验收结果、说明风险、给下一步。

## 当前版本

Forge 2.2 是 **安全任务单 / 省 token 安全执行版**。

核心原则：

- 默认小步执行。
- 先定义上下文预算，避免过度读取资料。
- 每轮停在复查关口，不自动无限推进。
- 跨工具使用只能由用户手动复制交接稿，Forge 不自动调度其他模型或产品。

## 真实反馈

早期版本已经能减少“AI 偷懒停下”，但在小任务里可能消耗过多 token。

Forge 2.2 把这个反馈转成产品改进：默认生成低 token 的安全任务单，明确上下文预算、停手条件、验收标准和复查关口。

## 最小演示

用户输入：

```text
我想做一个能自动整理会议纪要的小工具，第一版怎么开始？
```

Forge 典型输出：

```text
功能目标：
把会议文字整理成结构化纪要，第一版只支持用户粘贴文本。

第一版范围：
1. 输入会议文字
2. 输出摘要、待办事项、负责人、截止时间
3. 支持复制 Markdown

暂时不做：
- 账号系统
- 多人协作
- 自动录音转写
- 日历或飞书集成

给当前 agent 的任务单：
- 目标：搭一个本地可运行的单页原型
- 输入：一段会议文本
- 输出：结构化 Markdown 纪要
- 限制：不接外部 API，不引入复杂后端
- 验收：能用示例文本生成摘要和待办清单
- 验证：运行项目并手动检查输出

下一步：
1. 先做可运行原型
2. 再补 README 和示例输入
3. 最后验收能不能给朋友试用
```

## 高级能力

高级能力默认不会打扰小任务。只有项目变复杂、要交付、要多角色协作、要长期维护时才需要它们。

- Existing Project Audit：接手已有工程前先自查。
- 双索引：生成给人看的 `FORGE_INDEX.md` 和给智能代理/脚本看的 `forge_index.json`。
- Forge Doctor：发布或稳定性总检。
- 多角色协作：当前智能代理内的多角色任务分工。
- 人工交接：用户手动复制给其他工具的交接稿。
